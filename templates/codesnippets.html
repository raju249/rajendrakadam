Linked List in C.

I have used Prof.David.J.Malan of Harvard University's distribution code to write this code.I will be explaining the part of code and the logic why we used this line of code below the code snippet.HAPPY CODING. At the end you will have a link to view the whole source code and keep it for you along with all the header files.

PS: I have put the code in parts to make it easy for you to understand.It should be written in one file.

Getting Started:

So here we have bunch of includes at the top i.e the header files.You may be familiar with many of them like stdio.h,stdlib.h and unistd.h I have some additional header files like cs50.h which has functions like GetInt() which does the same work as scanf in our TurboC++.

Next is we have defined a struct to create a node for linked list.I have used typedef so that this would create a new datatype of type node and I would use it everywhere.So instead of struct node I would say it as node p; This is just creating a new datatype and I call it as node.

Next we have the head of the node and I named it as "first" which currently points to NULL as their is no list in memory.Note:first is a pointer variable which is a pointer to node.Then we have a couple of functions prototypes defined at the bottom of the code.They do the same work as their name suggests.

Main:

Now we come to main function which takes no argument.The code should be self-explanatory of what it does.It has a Menu of what the user has to do with the list aka delete a member,insert a member,search a member,traverse the list that is just look through the list and quit the program i.e erase the whole list from the memory and exit the program. This is written inside the do-while loop and keeps executing until you type "0" to quit.

The GetInt() function from cs50.h header file waits till the user enter a number."scanf" can also be used for the same.

Remember the "first" variable of type node we defined in the first part.We used the same variable and since it is a pointer variable which stores the address of the node.We assign the value (address) stored in first to predptr which is again a pointer to node.Well this part of code runs when you enter "0" to quit because we need to free memory before exiting the program.You should always free the memory if you use malloc to get some memory otherwise it may lead to memory leaks which makes your computer slow.

Next we check whether the list is empty if yes we straight away exit the program otherwise the body of the loop is executed.Try to understand the body of the while loop and see what it does.

"free" is used to free the memory and erase the memory used by your program.

ptr = ptr->next says that go to variable called "next" which is stored at the address stored in "ptr".Now we know that "next" is the part of the struct node and also stores address of the next node. free(predptr) frees the memory.

Traversing:

Now in this part and coming parts we will see all the implementation of functions for this list.First is "traverse" which traverses the list and prints the number in the list.

We always start from the head of the list so we always use the variable named "first" to start with our work further.

Next we check whether the first is NULL.If it is NULL then it says that the list is empty.If it is not NULL then print the number or the data stored in each node and we update the variable "ptr" to point to next node. sleep(time in milliseconds) just pauses your program for the time specified in the prantheses.

Inserting:

Now we see the longest function of inserting a new element in linked list.

This is function is long because we also sort the list in ascending order.But it is good to implement sorting in the list as many people don't do it.It helps you understand how pointers work.

Before reading further I suggest you to read the code and understand it yourself

Now we enter the function and create a new node pointer which is called newptr which stores the address a new node created.Here we use malloc to get a piece of memory of size of node.Now sometimes if your computer is out of memory then malloc returns NULL.So in the next line it is a just a sanity check whether we have memory or not.If we don't have memory we just return and stop further execution.But if we have memory we proceed further.Next we get the number from the user and insert in the linked list.

There are enough comments in the code to see what each line does.Because if I start explaining each and every line you'll keep scrolling down.But if you don't get what it does message me on WhatsApp or email me I would be happy to help.HAPPY CODING!

Searching in List:

In this part we would be seeing how is search function implemented.

Searching through list is the easiest part.You create a temporary pointer variable to search through the list and get updated if necessary to move to the next pointer.I get the number from the user and check the data or number stored at what ptr is pointing to.If I get that number I print it to the screen and come out of the loop or I update the pointer to point it to next node.That's it!!

Deleting data from list:

Last function that is delete a number or data from linked list.

Now deleting invloves a little bit of imagination.I suggest you to take pen/pencil and paper and create a list consisting of boxes that stores data and pointer to next node.Now try to delete or remove the node that you want but make sure that you don't loose reference through the remaining list.Follow the code line by line in your example and delete the node.You will understand how it is done.
